# OverlayNotepad

> 화면 위에 항상 떠 있는 **투명 메모장**.
> 배경을 투명하게 만들어 뒤의 콘텐츠를 보면서 메모할 수 있는 Windows 오버레이 도구입니다.

```
┌─────────────────────────────────────┐
│  [드래그 영역 - 상단 12px 바]        │  ← 드래그로 이동
├─────────────────────────────────────┤
│                                     │
│   투명 배경 위에 텍스트 표시         │
│   (테두리 + 그림자 효과 적용)        │
│                                     │
│   여기에 메모를 입력합니다...        │
│                                     │
└─────────────────────────────────────┘
       ↑ 테두리 드래그로 리사이즈
```

## 왜 만들었나

듀얼 모니터가 없는 환경에서 API 문서를 보면서 코딩하거나, 강의 화면 위에 필기하거나, 원문을 보면서 번역 메모를 남기고 싶을 때 — **화면 위에 투명하게 떠 있는 메모장**이 필요합니다.

기존 메모장은 배경이 불투명해서 뒤의 콘텐츠를 가리고, 스티키 노트는 투명도 조절이 안 됩니다. OverlayNotepad는 배경을 완전히 투명하게 만들어 **뒤의 화면을 그대로 보면서 메모**할 수 있습니다.

## 다운로드 및 실행

### 설치 방법

1. [GitHub Releases 페이지](https://github.com/rynnkitty/OverlayNotepad/releases)에서 최신 버전의 `OverlayNotepad.exe`를 다운로드합니다.
2. 다운로드한 EXE 파일을 원하는 위치에 복사합니다. (예: `C:\Tools\OverlayNotepad\`)
3. EXE 파일을 더블클릭하면 바로 실행됩니다.

> **설치 과정이 필요 없습니다.** 단일 EXE 파일로 동작하며, 레지스트리를 거의 건드리지 않습니다. 삭제는 EXE 파일을 지우는 것으로 충분합니다.

### Windows SmartScreen 경고가 뜰 때

코드 서명이 없어 "Windows의 PC 보호" 경고가 표시될 수 있습니다.

1. 경고 창의 **자세히 →** 를 클릭합니다.
2. **실행** 버튼을 클릭합니다.

## 빠른 시작 (30초)

1. `OverlayNotepad.exe` 실행 → 화면 중앙에 반투명 메모장이 나타납니다.
2. 바로 **텍스트를 입력**하세요. (포커스가 자동으로 잡혀 있습니다.)
3. 상단 **얇은 회색 바를 드래그**하여 원하는 위치로 이동합니다.
4. 윈도우 **테두리를 드래그**하여 크기를 조절합니다.
5. 작성한 내용은 **2초 후 자동 저장**됩니다. 앱을 종료했다가 다시 열어도 그대로 유지됩니다.
6. 메모장 내부를 **우클릭**하면 모든 설정 메뉴가 열립니다.

## 상세 사용 매뉴얼

### 1. 윈도우 이동 및 크기 조절

- **이동**: 메모장 **상단 12px의 얇은 바**를 왼쪽 버튼으로 누른 채 드래그합니다. 마우스를 바에 올리면 살짝 밝아져 드래그 영역을 확인할 수 있습니다.
- **크기 조절**: 윈도우 **네 모서리 또는 가장자리**에 마우스를 가져가면 리사이즈 커서로 바뀝니다. 그 상태에서 드래그하여 크기를 변경합니다.
- **최소 크기**: 가로 200px × 세로 150px (그보다 작아지지 않습니다.)
- **위치/크기 기억**: 종료 시 위치와 크기가 저장되어 다음 실행 시 복원됩니다. 저장된 위치가 화면 밖이면 자동으로 기본 위치(화면 중앙)로 복원됩니다.

### 2. 텍스트 입력과 저장

- 메모장이 열리면 **텍스트 박스에 자동 포커스**되어 바로 타이핑할 수 있습니다.
- **여러 줄 입력**: Enter 키로 줄 바꿈, Tab 키로 들여쓰기 (AcceptsReturn / AcceptsTab 활성).
- **표준 편집 기능**:
  - Ctrl+C / Ctrl+V / Ctrl+X : 복사 / 붙여넣기 / 잘라내기
  - Ctrl+A : 전체 선택
  - Ctrl+Z / Ctrl+Y : 실행 취소 / 다시 실행
  - Home / End / PageUp / PageDown : 이동
- **한글/IME 지원**: WPF 표준 TextBox 기반이므로 한국어/일본어/중국어 IME 모두 정상 동작합니다.
- **자동 저장**: 텍스트가 변경되면 **2초 디바운스** 후 자동 저장됩니다. 저장 직후 우클릭 메뉴에 **"자동 저장됨"**이 표시됩니다.
- **비정상 종료 대응**: 프로세스가 강제 종료되어도 마지막 저장 시점까지의 내용이 보존됩니다.

### 3. 투명도 조절

우클릭 메뉴 → **투명도** 서브메뉴에서 20% / 40% / 60% / 80% / 100% 중 선택합니다.

| 투명도 | 용도 |
|-------|------|
| 20% | 거의 보이지 않음. 얇게 깔아두고 싶을 때 |
| 40% | 매우 흐림. 참조용 |
| 60% | 중간. 일반적인 오버레이 메모 |
| **80%** | **기본값.** 텍스트는 잘 보이면서 뒤 콘텐츠도 확인 가능 |
| 100% | 완전 불투명 |

### 4. 배경 투명 / 전체 투명 모드

두 가지 투명 방식을 지원합니다. 우클릭 메뉴 → **배경 투명 모드** 토글로 전환합니다.

- **배경 투명 모드 (기본, 체크 상태)**
  - 배경만 투명하고 **텍스트는 불투명**하게 유지됩니다.
  - 뒤 콘텐츠를 보면서도 메모가 선명하게 보여야 할 때 사용합니다.
- **전체 투명 모드 (체크 해제 상태)**
  - 배경과 텍스트 모두 Window.Opacity에 따라 함께 투명해집니다.
  - 조용히 깔아두는 읽기 전용 참고 메모에 적합합니다.

### 5. Always on Top (항상 위에)

- 우클릭 메뉴 → **항상 위에** 토글 (기본: 켜짐 ✔).
- 켜져 있으면 다른 어떤 윈도우를 클릭해도 메모장이 항상 최상위에 표시됩니다.
- 트레이 아이콘 우클릭 메뉴에서도 동일하게 토글할 수 있습니다.

### 6. 테마 변경 (다크/라이트)

우클릭 메뉴의 **"테마: 다크 → 라이트"** 또는 **"테마: 라이트 → 다크"** 항목을 클릭하면 테마가 전환됩니다.

| 테마 | 글자색 | 테두리색 | 권장 상황 |
|------|--------|----------|-----------|
| **다크 (기본)** | 흰색 계열 | 검정 | 어두운 앱/에디터 위에서 사용할 때 |
| 라이트 | 검정 계열 | 흰색 | 밝은 앱/웹페이지 위에서 사용할 때 |

테마를 전환하면 글자색, 배경 톤, 테두리/그림자 색상이 함께 변경됩니다.

### 7. 글꼴 · 크기 · 색상 변경

우클릭 메뉴에서 다음 항목을 통해 서식을 바꿀 수 있습니다. 변경은 **전체 텍스트에 일괄 적용**됩니다. (부분 서식은 지원하지 않습니다.)

- **글꼴**: 시스템에 설치된 글꼴 목록에서 선택합니다. (메뉴를 열면 한 번 로딩 후 캐싱됩니다.)
- **글자 크기**: 프리셋(10 / 12 / 14 / 16 / 20 / 24 / 32 pt) 또는 **직접 입력…** 메뉴로 임의 값 지정.
- **글자 색상**: 색상 팔레트에서 선택합니다. 다크/라이트 테마에 따라 기본 팔레트가 달라집니다.

### 8. 텍스트 테두리와 그림자

투명 배경에서는 뒤 콘텐츠 색상이 무엇이든 글자가 잘 보여야 합니다. 두 가지 효과를 함께/따로 켤 수 있습니다.

- **텍스트 테두리 (기본 켜짐 ✔)**: 글자 외곽선에 아웃라인을 그려 가독성을 높입니다. `FormattedText + Geometry` 기반 커스텀 렌더링.
- **테두리 색상**: 우클릭 메뉴 → **테두리 색상**에서 직접 지정할 수 있습니다.
- **텍스트 그림자 (기본 켜짐 ✔)**: 글자에 DropShadow를 적용해 경계를 강조합니다.

> 권장: 밝은 배경 위에 올릴 땐 **라이트 테마 + 검정 테두리**, 어두운 배경 위에 올릴 땐 **다크 테마 + 검정 테두리**(기본값) 조합이 가장 잘 보입니다.

### 9. Click-Through (클릭 통과) 모드

메모장을 화면에 띄워둔 상태로 **뒤의 앱을 자유롭게 조작**하고 싶을 때 사용합니다.

- **켜는 방법 1**: 글로벌 핫키 `Ctrl + Shift + T`
- **켜는 방법 2**: 시스템 트레이 아이콘 우클릭 → **Click-Through** 체크
- **끄는 방법**: 동일한 핫키 또는 트레이 메뉴에서 다시 클릭
- **켜져 있을 때의 표시**:
  - 메모장 외곽에 **빨간색 점선 테두리**가 나타납니다.
  - 상단 드래그 바에 **`CLICK-THROUGH`** 라벨이 표시됩니다.
  - 트레이 아이콘의 툴팁이 `OverlayNotepad (Click-Through)`로 바뀝니다.
- **주의**: Click-Through가 켜져 있으면 마우스가 메모장을 통과하기 때문에 **우클릭 메뉴를 열 수 없습니다**. 반드시 핫키나 트레이 메뉴로 해제해야 합니다.
- **핫키 등록 실패 시**: 다른 앱이 `Ctrl+Shift+T`를 이미 잡고 있으면 시작 시 알림이 뜨고, 트레이 메뉴의 Click-Through 항목이 **`Click-Through (핫키 없음)`**으로 비활성화됩니다.

### 10. 글로벌 핫키

앱 포커스와 무관하게 어디에서나 동작합니다.

| 핫키 | 동작 |
|------|------|
| `Ctrl + Shift + N` | 메모장 표시 / 숨김 토글 |
| `Ctrl + Shift + T` | Click-Through 모드 토글 |

다른 앱이 같은 조합을 선점하고 있으면 등록이 실패하며, 시작 시 실패한 항목이 알림으로 표시됩니다.

### 11. 시스템 트레이

- 실행 중에는 **작업 표시줄 오른쪽 트레이에 아이콘**이 항상 표시됩니다.
- **더블클릭**: 메모장 표시/숨김 토글 (`Ctrl+Shift+N`과 동일).
- **우클릭 메뉴**:
  - 표시 / 숨김
  - 항상 위에 (체크 상태 동기화)
  - Click-Through (체크 상태 동기화, 핫키 등록 실패 시 비활성화)
  - 종료
- 창을 최소화하면 작업 표시줄이 아닌 **트레이로 숨겨집니다**. 다시 표시하려면 트레이 아이콘을 더블클릭하거나 `Ctrl+Shift+N`을 누르세요.

### 12. 컨텍스트 메뉴 (우클릭)

메모장 영역 어디에서나 마우스 우클릭으로 전체 설정 메뉴를 열 수 있습니다. 구성은 다음과 같습니다.

```
┌────────────────────────────┐
│ 투명도             ▶        │  → 20% / 40% / 60% / 80% / 100%
├────────────────────────────┤
│ ☑ 배경 투명 모드           │
│ ☑ 항상 위에                │
├────────────────────────────┤
│ 테마: 다크 → 라이트        │
│ 글꼴               ▶        │  → 시스템 설치 글꼴 목록
│ 글자 크기          ▶        │  → 10 / 12 / 14 / 16 / 20 / 24 / 32 / 직접 입력…
│ 글자 색상          ▶        │  → 색상 팔레트
├────────────────────────────┤
│ ☑ 텍스트 테두리            │
│ 테두리 색상        ▶        │
│ ☑ 텍스트 그림자            │
├────────────────────────────┤
│ 자동 저장됨 (표시용)       │
├────────────────────────────┤
│ ☐ Click-Through            │
│ 최소화                     │
│ 종료                       │
├────────────────────────────┤
│ kimkitty.net               │  → 개발자 웹사이트 열기
└────────────────────────────┘
```

## 설정 파일 및 데이터 위치

모든 설정과 메모 내용은 다음 경로에 JSON 형식으로 저장됩니다.

```
%AppData%\OverlayNotepad\settings.json
```

탐색기 주소창에 `%AppData%\OverlayNotepad` 를 입력하면 직접 열 수 있습니다.

### 설정 기본값

| 항목 | 기본값 |
|------|--------|
| 윈도우 위치 (X, Y) | 화면 중앙 |
| 윈도우 크기 (W × H) | 400 × 300 |
| 투명도 | 80% |
| 투명 모드 | 배경 투명 |
| Always on Top | 켜짐 |
| Click-Through | 꺼짐 |
| 테마 | 다크 |
| 글꼴 / 크기 | 맑은 고딕 / 14pt |
| 글자 색상 | `#FFFFFF` (다크) / `#000000` (라이트) |
| 텍스트 테두리 | 켜짐 |
| 테두리 색상 | `#000000` (다크) / `#FFFFFF` (라이트) |
| 텍스트 그림자 | 켜짐 |

### 초기화 방법

`settings.json` 파일을 삭제하면 다음 실행 시 기본값으로 복원됩니다. 설정 파일이 손상된 경우에도 자동으로 기본값으로 초기화됩니다.

### 백업 / 동기화

`settings.json` 파일만 복사/복원하면 됩니다. 메모 내용도 이 파일에 함께 저장되므로 다른 PC로 옮길 때도 이 파일만 복사하면 그대로 사용할 수 있습니다.

## 문제 해결 (FAQ)

**Q. 실행하면 SmartScreen 경고가 뜹니다.**
A. 코드 서명이 없어 발생합니다. **자세히 → 실행** 순으로 진행하면 됩니다. 이후 실행부터는 경고가 나타나지 않습니다.

**Q. `Ctrl+Shift+N` 또는 `Ctrl+Shift+T` 핫키가 동작하지 않습니다.**
A. 다른 앱이 같은 조합을 먼저 등록한 경우 실패합니다. 시작 시 알림이 뜨며, 충돌 앱(예: 일부 IME, 화면 캡처 도구 등)을 종료하고 OverlayNotepad를 재시작하면 정상 등록됩니다.

**Q. Click-Through를 켰는데 우클릭 메뉴가 안 열립니다.**
A. Click-Through 모드에서는 마우스가 메모장을 통과하므로 **의도된 동작**입니다. `Ctrl+Shift+T` 핫키 또는 트레이 아이콘 우클릭 → **Click-Through** 체크 해제로 복귀하세요.

**Q. 메모 내용이 사라졌습니다.**
A. 텍스트 변경 후 2초 디바운스 후 저장되므로, 변경 직후 1~2초 이내에 강제 종료하면 마지막 변경분이 반영되지 않을 수 있습니다. `%AppData%\OverlayNotepad\settings.json`을 열어 이전 내용을 확인할 수도 있습니다.

**Q. 창이 화면 밖으로 나가서 안 보입니다.**
A. 앱을 종료한 뒤 `settings.json`의 `WindowX`, `WindowY` 값을 지우거나, 파일을 통째로 삭제한 후 재실행하세요. 다음 실행 시 화면 범위 보정 로직이 작동하여 기본 위치로 복원됩니다.

**Q. 글자가 너무 흐려서 안 보입니다.**
A. 우클릭 → **투명도**를 100%로 올리거나, **텍스트 테두리**와 **텍스트 그림자**를 모두 켜고 테두리 색상을 배경 콘텐츠와 반대 톤으로 설정하세요.

**Q. 완전히 삭제하고 싶습니다.**
A. (1) 앱 종료 → (2) `OverlayNotepad.exe` 삭제 → (3) `%AppData%\OverlayNotepad` 폴더 삭제. 레지스트리에는 흔적이 거의 남지 않습니다.

**Q. 윈도우가 마우스 호버에만 보이고 클릭이 안 됩니다.**
A. Click-Through 모드가 켜진 상태입니다. 빨간 점선 테두리와 상단의 `CLICK-THROUGH` 라벨이 표시되어 있을 것입니다. `Ctrl+Shift+T`로 해제하세요.

## 빌드 방법 (개발자용)

직접 소스를 빌드하려면 아래 환경이 필요합니다.

### 사전 요구사항

- Windows 10 (1903+) 또는 Windows 11
- **Visual Studio 2019 이상** 또는 **.NET Framework 4.8 Developer Pack**
- Git

### 빌드 순서

```bash
git clone https://github.com/rynnkitty/OverlayNotepad.git
cd OverlayNotepad
```

**Visual Studio 사용 시**

1. `OverlayNotepad.sln`을 Visual Studio로 엽니다.
2. 빌드 구성을 `Release | Any CPU`로 설정합니다.
3. **빌드 → 솔루션 빌드** (`Ctrl+Shift+B`).
4. 산출물: `src/OverlayNotepad/bin/Release/OverlayNotepad.exe`

**MSBuild CLI 사용 시**

```bash
msbuild OverlayNotepad.sln /p:Configuration=Release /p:Platform="Any CPU"
```

### 프로젝트 구조

```
src/OverlayNotepad/
├── App.xaml / App.xaml.cs          # 앱 엔트리
├── MainWindow.xaml / .cs           # 메인 윈도우 (투명 오버레이)
├── Controls/                       # OutlinedTextControl (아웃라인 텍스트 렌더링)
├── Services/
│   ├── SettingsManager.cs          # JSON 설정 저장/로드
│   ├── AutoSaveManager.cs          # 디바운스 자동 저장
│   ├── ThemeManager.cs             # 다크/라이트 테마
│   ├── HotkeyService.cs            # 글로벌 핫키 (RegisterHotKey)
│   ├── ClickThroughService.cs      # WS_EX_TRANSPARENT 토글
│   └── TrayIconManager.cs          # 시스템 트레이 (WinForms NotifyIcon)
├── Helpers/                        # FontHelper / JsonHelper / ScreenHelper
├── Interop/                        # Win32 API 인터롭 정의
└── Resources/                      # 아이콘, 리소스
```

## 기술 스택

| 항목 | 선택 |
|------|------|
| 프레임워크 | .NET Framework 4.8 |
| UI | WPF (XAML + 코드비하인드) |
| 투명 윈도우 | `AllowsTransparency=True` + `WindowStyle=None` + `Background=Transparent` |
| 텍스트 효과 | `FormattedText` + `Geometry` (아웃라인), `DropShadowEffect` (그림자) |
| Click-Through | Win32 `WS_EX_TRANSPARENT` 인터롭 |
| 글로벌 핫키 | Win32 `RegisterHotKey` 인터롭 |
| 시스템 트레이 | WinForms `NotifyIcon` 인터롭 |
| 설정 저장 | JSON (수동 파싱, 외부 NuGet 의존성 없음) |
| DPI 인식 | 앱 매니페스트에서 Per-Monitor V2 선언 |

**외부 라이브러리 없음.** .NET Framework 기본 제공 라이브러리만 사용하여 **단일 EXE**로 배포합니다.
