# PRD - OverlayNotepad (투명 메모장)

## 프로젝트 개요

- **프로젝트명**: OverlayNotepad (투명 메모장)
- **목표**: 화면 위에 항상 떠 있는 투명 메모장. 배경을 투명하게 하여 뒤의 콘텐츠를 보면서 메모할 수 있는 오버레이 도구
- **대상 사용자**: 듀얼 모니터가 없거나, 화면 위에 메모를 띄워두고 참고하며 작업하는 사용자 (개발자, 학생, 번역가, 실시간 참고자료 필요한 직군)
- **플랫폼**: Windows 10/11 (오프라인 전용, 네트워크 불필요)

---

## 기술 스택

| 항목 | 선택 |
|------|------|
| 프레임워크 | .NET Framework 4.8 |
| UI | WPF (XAML + 코드비하인드) |
| 투명 구현 | WPF 네이티브 (WindowStyle=None, AllowsTransparency=True, Background=Transparent) |
| 텍스트 렌더링 | WPF TextBox + DropShadowEffect / 커스텀 FormattedText (테두리 효과) |
| 배포 | 단일 EXE (설치 불필요, .NET 4.8은 Windows 10+에 기본 포함) |

---

## 핵심 기능

### F1. 투명 배경

- **배경 투명 모드**: WPF AllowsTransparency=True + Background=Transparent로 배경만 투명 처리, 텍스트는 불투명 유지
- **전체 투명도 모드**: Window.Opacity를 활용한 전체(배경+텍스트) 투명도 조절
- **모드 전환**: 컨텍스트 메뉴에서 "배경 투명" / "전체 투명" 모드 토글 전환
- **투명도 조절**: 우클릭 컨텍스트 메뉴 내 슬라이더로 0%~100% 투명도 조절

### F2. Always on Top

- 항상 최상위 표시 (Topmost) 토글 기능
- 컨텍스트 메뉴 및 글로벌 핫키로 on/off 전환
- 기본값: 활성화

### F3. 텍스트 테두리 및 그림자

- **텍스트 아웃라인**: WPF FormattedText + Geometry를 사용하여 글자에 테두리 효과 적용 (커스텀 컨트롤)
- **그림자 효과**: DropShadowEffect를 활용한 그림자로 가독성 강화
- **테두리 색상 선택**: 사용자가 테두리/그림자 색상을 직접 선택 가능
- **토글**: 테두리+그림자 효과를 켜기/끄기 가능
- **목적**: 어떤 배경 위에서든 텍스트 가독성 확보

### F4. 서식 지원 (전체 적용)

- **글꼴 변경**: 시스템에 설치된 글꼴 중 선택
- **글자 크기**: 자유롭게 크기 조절
- **글자 색상**: 색상 선택기로 텍스트 색상 변경
- **적용 범위**: 전체 텍스트에 일괄 적용 (부분 서식 미지원)

### F5. 다크/라이트 테마

- **다크 테마**: 글자색 흰색/밝은 색, 배경 검정 반투명 (기본값)
- **라이트 테마**: 글자색 검정/어두운 색, 배경 흰색 반투명
- **테마 전환**: 컨텍스트 메뉴에서 토글로 즉시 전환
- 테마 전환 시 글자색, 배경색, 테두리색이 함께 변경

### F6. Click-Through (클릭 통과) 토글

- **활성화 시**: 마우스 클릭이 메모장을 통과하여 뒤의 앱에 전달 (Win32 WS_EX_TRANSPARENT 인터롭)
- **비활성화 시**: 일반 윈도우처럼 클릭이 메모장에 잡힘
- **전환 방법**: 글로벌 핫키 또는 트레이 메뉴로만 전환 (click-through 상태에서는 우클릭 메뉴 사용 불가)
- **시각적 표시**: click-through 모드 시 테두리 색상 변경 등으로 상태 표시

### F7. 자동 저장

- 텍스트 내용을 로컬에 자동 저장 (AppData 폴더 내 설정 파일)
- 프로그램 종료 후 재실행 시 이전 내용 자동 복원
- 별도 파일 열기/저장 기능 없음 (단일 메모 용도)
- 저장 주기: 텍스트 변경 후 일정 시간(예: 2초) 디바운스

### F8. 윈도우 관리

- **타이틀바 없음**: WindowStyle=None, 커스텀 드래그 영역으로 이동
- **자유 리사이즈**: 윈도우 테두리 드래그로 크기 조절 (커스텀 리사이즈 핸들 또는 WindowChrome)
- **위치/크기 기억**: 종료 시 위치와 크기를 저장, 다음 실행 시 복원
- **화면 범위 보정**: 저장된 위치가 현재 화면 밖이면 기본 위치로 복원

### F9. 시스템 트레이

- 트레이에 항상 아이콘 표시 (WinForms NotifyIcon 인터롭 또는 Hardcodet.NotifyIcon.Wpf)
- 최소화 시 트레이로 숨김
- 트레이 아이콘 더블클릭으로 복원
- 트레이 우클릭 메뉴: 표시/숨김, Always on Top, Click-Through, 종료

### F10. 글로벌 핫키

- **표시/숨김 토글**: Ctrl+Shift+N (기본값, 변경 가능 고려)
- **Click-Through 토글**: Ctrl+Shift+T (기본값)
- 다른 앱 사용 중에도 핫키로 메모장 제어 가능
- RegisterHotKey Win32 API 인터롭 사용

### F11. 컨텍스트 메뉴 (우클릭)

포함 항목:
- 투명도 슬라이더
- 배경 투명 모드 전환
- Always on Top 토글
- Click-Through 토글
- 테마 전환 (다크/라이트)
- 글꼴/크기/색상 설정
- 텍스트 테두리/그림자 토글 및 색상
- 자동저장 상태 표시
- 최소화
- 종료

## 비기능 요구사항

### 성능
- 시작 시간: 2초 이내 (WPF 초기화 포함)
- 메모리 사용량: 80MB 이하 (유휴 상태, WPF 런타임 포함)
- CPU 사용: 유휴 시 0%에 가까울 것
- 자동 저장: 텍스트 변경 후 2초 디바운스

### 안정성
- 비정상 종료 시에도 자동 저장 데이터 보존
- 설정 파일 손상 시 기본값으로 복원

### 호환성
- Windows 10 (1903+) / Windows 11
- .NET Framework 4.8 (OS 기본 포함)
- DPI 인식: WPF 기본 DPI 스케일링 활용 (별도 멀티 모니터 DPI 처리 없음)
- 다중 모니터: 위치 저장 시 화면 범위 보정만 처리

### 배포
- 단일 EXE 파일로 배포
- 설치/언설치 프로세스 없음
- 레지스트리 수정 최소화 (글로벌 핫키 등록용만)

## 제약 조건

- **기술 스택**: .NET Framework 4.8 + WPF (변경 불가)
- **네트워크**: 완전 오프라인 동작 (업데이트 체크, 텔레메트리 등 일체 없음)
- **파일 관리**: 단일 메모만 지원 (멀티 탭/멀티 메모 미지원)
- **서식**: 전체 텍스트에 일괄 적용만 지원 (부분 서식 미지원)
- **외부 라이브러리**: NuGet 의존성 최소화 (단일 EXE 배포를 위해). 시스템 트레이는 WinForms NotifyIcon 인터롭 또는 내장 구현 우선

## 기술적 고려사항 및 트레이드오프

### WPF 투명 윈도우의 텍스트 입력
- WPF AllowsTransparency=True 설정 시 표준 TextBox가 투명 배경 위에서 정상 동작
- **장점**: GDI+ 커스텀 에디터 불필요, IME/한글 입력 자동 지원, 표준 편집 기능(Undo, 복사/붙여넣기 등) 내장
- **주의**: AllowsTransparency=True 시 하드웨어 가속이 비활성화됨 → 소프트웨어 렌더링으로 전환. 일반 메모장 용도에서는 성능 영향 미미

### 텍스트 테두리 효과 구현 방식
- **방식 A**: 커스텀 컨트롤에서 FormattedText + Geometry로 텍스트 아웃라인 직접 렌더링
- **방식 B**: TextBox 위에 오버레이 레이어를 두어 테두리 효과만 별도 렌더링
- **결정**: 방식 A 채택 (테두리 효과 ON 시 커스텀 렌더링, OFF 시 일반 TextBox)
- **트레이드오프**: 테두리 모드에서 커스텀 렌더링 복잡도 vs 완전한 아웃라인 제어

### 배경 투명 모드 vs 전체 투명 모드
- 배경 투명 모드: AllowsTransparency + Background=Transparent → 배경만 투명, 텍스트 불투명
- 전체 투명 모드: Window.Opacity 조절 → 전체가 함께 투명해짐
- **결정**: 두 모드 모두 지원하여 사용자가 상황에 맞게 선택

### Click-Through 모드의 제어
- Click-Through 활성화 시 메모장 자체에 마우스 이벤트가 전달되지 않음
- WPF에서는 Win32 인터롭으로 WS_EX_TRANSPARENT 윈도우 스타일 동적 변경
- **해결**: 트레이 메뉴 또는 글로벌 핫키로만 모드 해제 가능하도록 설계

### 시스템 트레이 (WPF 제약)
- WPF에는 네이티브 NotifyIcon이 없음
- **선택지**: WinForms NotifyIcon을 System.Windows.Forms 참조 추가하여 사용 (가장 간단)
- 외부 NuGet 패키지(Hardcodet.NotifyIcon.Wpf) 대신 WinForms 인터롭으로 단일 EXE 유지

## 설정 저장 항목

모든 설정은 `%AppData%/OverlayNotepad/settings.json`에 저장:

| 항목 | 기본값 |
|------|--------|
| 윈도우 위치 (X, Y) | 화면 중앙 |
| 윈도우 크기 (W, H) | 400x300 |
| 투명도 | 80% |
| 투명 모드 | 배경 투명 |
| Always on Top | true |
| Click-Through | false |
| 테마 | 다크 |
| 글꼴 | 맑은 고딕 |
| 글자 크기 | 14pt |
| 글자 색상 | #FFFFFF (다크) / #000000 (라이트) |
| 텍스트 테두리 | 활성화 |
| 테두리 색상 | #000000 (다크) / #FFFFFF (라이트) |
| 메모 내용 | (빈 문자열) |
