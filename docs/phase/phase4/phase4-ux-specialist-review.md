# Phase 4 검토 리포트 — 한유엑 (UX 전문가)

> 검토일: 2026-04-17
> 검토 대상: Phase 4 아키텍처 초안 (글로벌 핫키 + Click-Through + 최종 마무리)

## 요약

- ✅ 핫키 체계 — 2개 핫키로 핵심 기능 커버, 적절한 조합
- ✅ WS_EX_TRANSPARENT 접근 방식 — Win32 표준, 안정적
- ⚠️ Click-Through 시각 피드백 설계 — 상태 전환이 명확하지 않으면 UX 실패
- ❌ Click-Through 진입/탈출 학습 곡선 — 처음 사용 시 "왜 클릭이 안 되지?" 혼란 불가피, 해결 방안 필요

## 항목별 검증 결과

### 1. 글로벌 핫키 인터랙션
- ✅ 포커스와 무관하게 동작 — 오버레이 도구의 핵심 UX
- ✅ 표시/숨김 토글 — 단일 핫키로 ON/OFF, 상태 기억이 간단
- ⚠️ 핫키 등록 실패 시 UX — 무반응이 아닌 명시적 알림 필요
  - 권고: 시작 시 핫키 등록 결과를 트레이 벌룬으로 1회 표시

### 2. Click-Through 시각 피드백 설계

현재 제안: "테두리 색상 변경"

**Before (일반 모드)**:
```
┌─────────────────────┐  ← 일반 테두리
│ [드래그 바]          │
│ 메모 텍스트          │
└─────────────────────┘
```

**After (Click-Through 모드) — 권고안**:
```
┌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┐  ← 빨간 점선 테두리
│ [CLICK-THROUGH] ▲   │  ← 상단 바에 상태 텍스트
│ 메모 텍스트          │
└╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┘
```

- 색상 단독 변경은 부족 — 색맹 사용자 고려, 패턴(점선) + 텍스트 + 색상 3중 신호
- 상단 드래그 바에 모드 텍스트 표시 — Click-Through에서는 드래그 불필요하므로 영역 재활용
- 트레이 아이콘에도 상태 반영 (아이콘 색상 또는 배지)

### 3. 상태 조합 매트릭스

| 배경 투명 | 전체 투명 | Click-Through | 시각적 결과 | UX 판단 |
|----------|----------|--------------|------------|---------|
| ON | OFF | OFF | 배경 투명, 텍스트 불투명, 클릭 가능 | ✅ 정상 |
| ON | OFF | ON | 배경 투명, 텍스트 불투명, 클릭 통과 | ✅ 핵심 사용 케이스 |
| OFF | ON (50%) | OFF | 전체 반투명, 클릭 가능 | ✅ 정상 |
| OFF | ON (50%) | ON | 전체 반투명, 클릭 통과 | ⚠️ 주의 — 반투명+클릭통과 |
| ON | ON (10%) | ON | 거의 보이지 않음, 클릭 통과 | ❌ 위험 — 사실상 제어 불가 |

- 권고: Click-Through 활성 시 전체 투명도 최소 20% 강제 (완전히 사라지는 것 방지)

### 4. DPI 인식
- ✅ app.manifest dpiAware 선언 — WPF는 기본적으로 DPI 인식, manifest 선언으로 보강
- ⚠️ Per-Monitor DPI (다중 모니터) — PRD에서 "별도 멀티 모니터 DPI 처리 없음"으로 명시, 수용 가능

## 파라미터 조정 권고

| 항목 | 원래 설계 | 권고값 | 근거 |
|------|----------|--------|------|
| Click-Through 피드백 | 테두리 색상만 | 빨간 점선 테두리 + 상단 바 텍스트 + 트레이 아이콘 변경 | 색상 단독은 인지 부족, 3중 신호 필요 |
| Click-Through 시 투명도 하한 | 없음 | 최소 20% (Window.Opacity >= 0.2) | 완전 투명 + Click-Through = 제어 불가 |
| Click-Through 최초 활성화 | 즉시 | 첫 1회 트레이 벌룬으로 해제 방법 안내 | 학습 곡선 완화 |
| 핫키 충돌 알림 | 미정 | 시작 시 트레이 벌룬 1회 표시 | 무반응 방지 |

## 리스크

1. **Click-Through "갇힘"**: 핫키를 모르는 사용자가 Click-Through에서 빠져나오지 못함
   - 대안: 트레이 메뉴가 유일한 탈출구 (트레이 아이콘은 Click-Through 영향 안 받음)
2. **다중 모니터 DPI 차이**: WPF 기본 스케일링으로 대부분 커버, 극단적 DPI 차이에서 텍스트 흐림 가능
   - 대안: Per-Monitor V2 DPI 인식은 백로그로 (PRD 범위 밖)
