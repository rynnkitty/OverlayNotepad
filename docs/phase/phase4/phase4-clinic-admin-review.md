# Phase 4 검토 리포트 — 김수진 (최종 사용자 관점)

> 검토일: 2026-04-17
> 검토 대상: Phase 4 아키텍처 초안 (글로벌 핫키 + Click-Through + 최종 마무리)

## 요약

- ✅ 글로벌 핫키 — 다른 앱 사용 중에도 메모장 제어 가능, 실용적
- ✅ Click-Through — 읽기 전용 오버레이로 참고자료 표시 시 매우 유용
- ⚠️ 상태 전환 혼란 — Click-Through ON/OFF 상태를 직관적으로 구분할 수 있어야 함
- ⚠️ 핫키 기억 부담 — 2개 핫키(Ctrl+Shift+N, Ctrl+Shift+T)는 적절한 수준

## 항목별 검증 결과

### 1. 일반 사용자 관점 핫키 사용성
- ✅ Ctrl+Shift+N — "N"ote의 연상 기억 가능
- ✅ Ctrl+Shift+T — "T"ransparent/Through의 연상 기억 가능
- ⚠️ 핫키 조합이 3키 동시 — 익숙하지 않은 사용자에게는 진입 장벽, 하지만 글로벌 핫키 특성상 불가피

### 2. Click-Through 상태 인지
- ⚠️ 테두리 색상 변경만으로 충분한가? — 미세한 색상 차이는 놓칠 수 있음
- 권고: 테두리 색상 + 점선 패턴 또는 아이콘 오버레이 등 복합적 시각 피드백

### 3. 배포/설치 경험
- ✅ 단일 EXE, 설치 불필요 — USB에 담아 이동하며 사용 가능, 매우 편리
- ✅ .NET 4.8 기본 포함 — 별도 런타임 설치 불필요

## 파라미터 조정 권고

| 항목 | 원래 설계 | 권고값 | 근거 |
|------|----------|--------|------|
| Click-Through 시각 피드백 | 테두리 색상 변경 | 테두리 색상(빨간색 계열) + 상단 바에 "CLICK-THROUGH" 텍스트 표시 | 색상만으로는 상태 인지가 약함 |
| 트레이 아이콘 상태 | 동일 아이콘 | Click-Through 시 아이콘 변경 또는 오버레이 | 트레이에서도 상태 구분 필요 |

## 리스크

1. **상태 혼란**: 사용자가 Click-Through 모드인지 모르고 클릭이 안 되는 상황 — 명확한 시각 피드백 필수
2. **복구 경로 학습**: Click-Through 해제 방법을 모르면 앱을 강제 종료하게 됨 — 트레이 메뉴 안내 또는 첫 사용 시 가이드
