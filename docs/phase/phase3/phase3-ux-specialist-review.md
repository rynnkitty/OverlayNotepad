## 항목별 검증 결과

### ✅ 통과

1. **컨텍스트 메뉴 계층 구조**: PRD F11의 전체 항목을 단일 메뉴에 평탄하게 배치하면 항목이 과다함. 구분선(Separator)으로 논리적 그룹핑하는 설계는 적절
2. **테마 전환 UX**: 토글 형식(다크 <-> 라이트)은 이 앱의 2가지 테마에 적합. 복잡한 테마 선택 UI 불필요
3. **트레이 아이콘 더블클릭 복원**: Windows 표준 동작 패턴을 따름. 사용자 학습 비용 0
4. **서식 전체 적용**: 범위가 명확하여 사용자 혼란 없음

### ⚠️ 주의

1. **컨텍스트 메뉴 깊이 제한**: 서브메뉴를 1단계 이하로 제한해야 함. 2단 이상 서브메뉴는 투명 오버레이 앱에서 조작이 불편
   - 권고: 글꼴/크기/색상은 서브메뉴 1단, 세부 선택은 다이얼로그로 위임

2. **투명도 조절 UX**: 슬라이더 대신 단계별 선택 시 "현재 선택된 투명도"를 시각적으로 표시해야 함 (체크마크 또는 라디오)

3. **테마 전환 시 시각적 피드백**: 전환 즉시 모든 요소가 변경되므로 깜빡임(flicker) 방지 필요. 색상 변경 순서를 배경 -> 텍스트 -> 테두리로 일괄 적용 권고

4. **트레이 메뉴 vs 컨텍스트 메뉴 역할 분리**: 두 메뉴의 항목이 중복되면 혼란. 역할 명확화 필요
   - 트레이 메뉴: 앱 수준 조작 (표시/숨김, Always on Top, 종료)
   - 컨텍스트 메뉴: 콘텐츠/서식 수준 조작 (투명도, 서식, 테마, 테두리)
   - 공통: 최소화, 종료

- **리스크**: 컨텍스트 메뉴 항목이 10개 이상이면 메뉴 자체가 화면 하단을 넘길 수 있음
- **대안**: 메뉴 항목을 그룹별 Separator로 구분하고, "설정..." 항목으로 고급 설정을 별도 창으로 분리하는 것을 Phase 4 Backlog에 추가
- **리스크**: 투명 배경 위의 컨텍스트 메뉴가 뒤 배경에 따라 읽기 어려울 수 있음
- **대안**: 컨텍스트 메뉴에는 불투명 배경 적용 (WPF ContextMenu 기본 동작으로 해결)
