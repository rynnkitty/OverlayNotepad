# Phase 2 검토 리포트 - 김수진 (일반 사용자 관점)

> **검토일**: 2026-04-17
> **검토 대상**: Phase 2 아키텍처 초안 (텍스트 테두리/그림자 + 자동 저장 + 윈도우 관리)
> **비고**: 이 프로젝트는 투명 메모장 유틸리티로, 김수진 페르소나를 "업무 중 메모장을 활용하는 일반 사용자" 관점으로 재해석하여 검토

---

## 2. 항목별 검증 결과

### 실사용 적합성
- 투명 메모장을 화면 위에 띄워두고 참고 자료를 보며 메모하는 시나리오에서 테두리/그림자는 필수
- 배경이 밝은 웹페이지 위에 흰색 글자를 띄우면 테두리 없이는 읽을 수 없음
- 자동 저장은 "저장 버튼을 누를 필요 없다"는 점에서 사용자 부담 최소화

### 데이터 안전성
- 비정상 종료 시 데이터 보존은 업무용 메모에서 매우 중요
- AppDomain.UnhandledException 핸들링으로 마지막 저장 보장은 적절

### 학습 비용
- 테두리/그림자는 기본 ON이므로 사용자가 별도 설정할 필요 없음
- 자동 저장도 자동이므로 학습 비용 0

## 3. 파라미터 조정 권고

| 항목 | 원래 설계 | 권고값 | 근거 |
|------|----------|--------|------|
| 테두리 기본 상태 | 활성화 | 활성화 유지 | PRD 명시, 투명 배경에서 테두리 없으면 가독성 문제 |
| 자동 저장 디바운스 | 2초 | 2초 유지 | 빠른 타이핑 중 저장 빈도와 데이터 유실 위험의 균형 |

---

## 4. 리스크 및 대안

### ⚠️ 자동 저장 실패 시 사용자 인지 부재
- 자동 저장이 실패해도 사용자가 알 수 없음 (Phase 2에는 상태 표시 UI 없음)
- 디스크 용량 부족, 파일 접근 권한 문제 등으로 저장 실패 가능
- **권고**: 저장 실패 시 최소한의 알림 (윈도우 타이틀 변경 또는 콘솔 로그)
- Phase 3에서 컨텍스트 메뉴의 "자동저장 상태 표시"로 해결 예정이므로 Phase 2에서는 내부 로그만 기록해도 충분
