## 2. 항목별 검증 결과

### 텍스트 가독성
- 투명 배경 위 텍스트는 뒤 배경에 따라 가독성이 크게 변함
- 테두리(아웃라인) + 그림자 조합이 가장 안정적인 가독성 확보 방법
- 다크 테마 기본값(흰색 글자 + 검정 테두리 + 검정 그림자)은 대부분의 배경에서 적절

### 테두리 효과 전환 UX
- 테두리 ON/OFF 전환 시 텍스트 레이아웃이 변경되면 사용자 혼란 발생 가능
- **Before**: 테두리 ON -> 커스텀 컨트롤, OFF -> TextBox 전환 시 위치/크기 변동
- **After**: 전환 시 텍스트 내용과 커서 위치가 보존되어야 함
- 조작 단계: 우클릭 > 테두리 토글 = 2단계 (적절)

### 자동 저장 UX
- 2초 디바운스는 타이핑 중 저장이 발생하지 않아 UX 적절
- 저장 상태 피드백이 필요하지만 Phase 3 컨텍스트 메뉴에서 "자동저장 상태 표시"로 예정
- Phase 2에서는 별도 시각적 피드백 불필요 (오버레이 메모장 특성상 최소 UI)

### 윈도우 관리 UX
- 위치/크기 복원은 "마지막 상태 그대로" 원칙으로 적절
- 화면 범위 보정은 다중 모니터 사용자에게 필수

## 3. 파라미터 조정 권고

| 항목 | 원래 설계 | 권고값 | 근거 |
|------|----------|--------|------|
| 테두리 두께 범위 | 미정 | 0.5~3.0px | 0.5px 단위로 세밀 조절, 4K 디스플레이에서 1px은 너무 얇을 수 있음 |
| 그림자 offset | 미정 | 1.5px (X,Y 동일) | 자연스러운 그림자 방향, 과도한 offset은 이중 텍스트처럼 보임 |
| 그림자 blur | 미정 | 3~5px | 3px이면 충분한 가독성, 5px 이상은 텍스트 뭉개짐 |
| 기본 윈도우 크기 | 400x300 | 400x300 유지 | PRD 명시값 |
| 최소 윈도우 크기 | 미정 | 200x150 | 이보다 작으면 텍스트 편집이 비실용적 |

## 4. 리스크 및 대안

### ⚠️ 커스텀 렌더링 <-> TextBox 전환 시 시각적 불연속
- 테두리 ON: 커스텀 FormattedText 렌더링 / OFF: 표준 TextBox
- 전환 시 글자 위치, 줄간격, 자간이 미세하게 다를 수 있음
- **권고**: TextBox를 항상 입력 레이어로 유지하고, 테두리 효과는 별도 오버레이 레이어에서 렌더링하는 구조 검토
- **대안**: 테두리 효과는 DropShadowEffect + Stroke 시뮬레이션(다중 DropShadow)으로 TextBox 자체에 적용

### ⚠️ 설정 파일 손상 시 사용자 인지
- 설정 파일이 손상되어 기본값으로 복원되면 사용자가 "이전 설정이 날아갔다"고 느낄 수 있음
- **권고**: 손상된 설정 파일을 `.bak`으로 백업 후 기본값 복원, 복원 사실을 다음 기회(Phase 3 트레이 등)에 알릴 수 있도록 플래그 저장
