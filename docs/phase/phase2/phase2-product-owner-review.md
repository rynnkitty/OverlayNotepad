## 2. 항목별 검증 결과

### MVP 범위
- F3(텍스트 테두리/그림자)는 PRD P0 필수 기능으로 Phase 2에 배치 적절
- F7(자동 저장), F8(윈도우 관리)는 P1 높음 기능으로 이 시점에 구현 타당
- 범위가 과도하지 않음 - 2개 Sprint로 소화 가능

### Sprint 분할
- Sprint 1(텍스트 효과)과 Sprint 2(설정/저장/윈도우)의 분리가 합리적
- Sprint 1이 기술 난이도가 높은 커스텀 렌더링이므로 먼저 배치한 것이 올바름
- Sprint 2는 Sprint 1의 설정값을 저장하므로 의존성 방향이 맞음

### 우선순위 정합성
- PRD 우선순위 P0(F3) + P1(F7, F8) 조합으로 로드맵과 일치

## 4. 리스크 및 대안

### ⚠️ 커스텀 렌더링 컨트롤의 편집 기능 호환성
- FormattedText + Geometry 방식은 텍스트를 "그리는" 것이므로 TextBox의 커서, 선택, IME 등 표준 편집 기능과 별개
- **권고**: 테두리 모드에서도 편집은 TextBox가 담당하고, 커스텀 렌더링은 표시 전용 레이어로 분리하는 설계를 Sprint 1에서 검증
- **폴백**: FormattedText가 편집과 양립 불가한 경우 DropShadowEffect만으로 가독성 확보 (그림자만으로도 80% 이상의 가독성 개선)
