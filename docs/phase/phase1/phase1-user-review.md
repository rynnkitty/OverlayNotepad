## 항목별 검증 결과

### 1. 즉시 사용 가능성 (⚠️ 주의)
투명 메모장은 "실행 즉시 타이핑"이 가능해야 한다. 실행 후 TextBox에 자동 포커스가 설정되어야 한다. 추가 클릭이 필요하면 사용성이 떨어진다.

**권고**: `FocusManager.FocusedElement` 또는 `Loaded` 이벤트에서 TextBox.Focus() 호출 필수.

### 2. 종료 방법 명확성 (⚠️ 주의)
타이틀바가 없으므로 닫기 버튼이 없다. Sprint 1에서는 컨텍스트 메뉴가 아직 없으므로, Alt+F4 외에 종료 방법이 없다. Sprint 1 단계에서 최소한의 종료 수단이 필요하다.

**권고**: Sprint 1에서도 최소한의 종료 방법(우클릭 > 종료 또는 닫기 버튼)을 제공하거나, Sprint 2에서 컨텍스트 메뉴로 해결될 때까지 Alt+F4가 동작하는지 확인.

### 3. 드래그 인지성 (⚠️ 주의)
드래그 영역이 투명 배경에서 시각적으로 구분되지 않으면, 사용자가 윈도우 이동 방법을 알 수 없다. 미세한 시각적 힌트(마우스 오버 시 반투명 색상 변경 등)가 필요하다.

### 4. 텍스트 입력 기본 동작 (✅ 통과)
WPF TextBox의 표준 기능(Ctrl+C/V/X, Ctrl+Z/Y, Ctrl+A)이 기본 제공되므로 별도 구현 불필요.

## 리스크

- Sprint 1 단계에서 종료 방법이 Alt+F4뿐이면, 일반 사용자가 앱을 닫지 못하는 상황이 발생할 수 있다. Sprint 1에서 최소한의 닫기 수단을 확보할 것.
